 In order to implement _build_num_op(), we need a utility method `find_op()`. The input is `tokens`, the same tokens list that is passed to
  _build_num_op(). It will attempt to find the first operand token of the lowest order of operator precedence. Earlier in this session, Claude defined the 
  operator precendence orders as follows:
  
    Operator precedence — three levels (highest to lowest)
    1. <power>
    2. <times> / <divide>
    3. <plus> / <minus>
  
  So we will start at Level 3 (<plus> / <minus>) looking for either of those two operators. If we find none, we look for Level 2, then Level 1. If we find 
  no operand token, this is an error and we will return `None`.
  
  After finding our operand token, we will copy the value of that token into a variable called `splitter: str`. Copy the tokens in `tokens[]` prior to the
  splitter token into a list called `left`, and copy the tokens following the splitter token into a list called `right`. If there are no tokens before the 
  splitter, or there are no tokens after the splitter, this is an error and we will return `None`.
