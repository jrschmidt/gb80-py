Help me plan how we can architect `_build_num_op()` and `_build_str_op()`. In this case, "op" refers to "operation", specifically, an operation on two terms such as `X + Y` or `"HELLO " + N$`.
  
What we will call an "op expression" in this app consists of two terms and an operand. We will reference these in a BasicLine object representing an op expression as follows:

line_object: BasicLine = {
    "op": {the token representing the operand},
    "term1": {the first term},
    "term1": {the first term}
}

For string op expressions, there is only one allowable binary operation, concatenation, represented by the plus sign character ('+') and the token "<concatenate>".

So the op expression for the BASIC line
400 T$ = "NAME IS " + T2$

would be built as

{
    "op-type": "<concatenate>",
    "term1": {
        "op": "<string_literal>",
        "string": "NAME IS "
    },
    "term2": {
        "op": "string_variable>",
        "variable": "T2"
    }
}
